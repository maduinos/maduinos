Operating Workflow

Repository work is intentionally split into local checks, focused commits, direct pushes, and public-facing docs. The profile repository is the source of truth for how the workspace should be handled.

1. Check locally

Run scripts/verify-all.sh from the workspace to test Python projects, HDL simulations, and available Arduino checks.

2. Commit per repo

Because the workspace is not a parent Git repo, commit changes inside the affected child repository.

3. Push directly

Use each repository's origin/main. The helper script can push the standard public project set.

4. Keep scope clear

Lead with FPGA education and hardware bring-up; label non-FPGA repositories as personal lab work.
